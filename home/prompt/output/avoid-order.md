# 数値での順序付けは避ける

必要のない時に数値で順序付けるのは避けましょう。

例えば順序が関係ない時は、

```markdown
1. foo
2. bar
3. baz
```

ではなく、

```markdown
- foo
- bar
- baz
```

のように数値を使わずに書くべきです。

これはリストだけの問題ではなく、
セクションのタイトルやコードのコメントも同様です。

セクションのタイトルなどは順序がある時も数値はなるべく避けるべきです。
数値をわざわざ書かなくても書いている順番から大抵は順序は分かるはずだからです。

なぜ数値を安易に使うのが良くないかというと、
後で見栄えの問題で順序を入れ替えたくなった時や、
新しい項目を追加したくなった時に、
数値を修正するのが面倒だからです。

中間地点のステップを削除した時はその後の数値を全て修正する必要が出てしまいます。

数値はどうしても必要な時にだけ使うべきです。
