# コマンドのガイドライン

## 非推奨

- `cat`: 代わりにあなた自身がファイルを読み込んでください。
- `find`: 代わりに自前の検索か`fd`を使ってください。
- `gh api`: 代わりに`gh`の専用サブコマンドを使ってください。
- `git commit`: 定型的ではないコミットは基本的に人間が行います。
- `grep`: 検索には`Search`や`rg`を使ってください。パイプのフィルタリングはなるべくせずに出力内容を全部読んでください。

- `head`: 全部読んでください。
- `tail`: 全部読んでください。

## 禁止

- `rm`: 代わりに`trash`を使ってください。
